# ek-unique-search-cli

A command-line tool for generating unique items, tagging them, and searching/persisting them locally. Useful for anyone needing easy unique item creation and fast local search.

## Project Structure

```
.
├── ek_unique_cli.py     # Main CLI entry point
├── cli.py               # Command/arg parsing, command dispatch
├── domain.py            # Item entity and business logic
├── storage.py           # JSON persistence
├── utils.py             # Helpers and validation
├── data/                # Local persistent storage for items
├── tests/               # Pytest test directory
├── .env.example         # Example environment variables
├── .gitignore           # Ignore pycache/ and data/
└── README.md            # This file
```

## Requirements

- Python 3.8+
- [pytest](https://pytest.org/) for running tests
- No non-stdlib dependencies for CLI (uses [python-dotenv](https://pypi.org/project/python-dotenv/) only for `.env` file support if desired, but also works without)

## Setup

1. **(Recommended)** Create and activate a virtual environment:

    ```bash
    python3 -m venv venv
    source venv/bin/activate
    ```

2. **Install optional dependencies** (for tests; Python CLI works out of the box):

    ```bash
    pip install pytest python-dotenv
    ```

3. **Configure environment variables**
    
    - Copy `.env.example` to `.env` and adjust, or export directly:
      ```bash
      export DATA_FILE=./data/items.json
      ```
    - By default, data will be stored at `./data/items.json`.
    - You can set `DATA_FILE` to any valid file path to store the items database elsewhere.

4. **Create data directory (if not yet exists):**

    The CLI will auto-create `./data/` as necessary, but you may want to `mkdir data/` if running in restricted environments.

## Usage

Run the CLI tool:

```bash
python ek_unique_cli.py --help
```

You should see:

```
ek-unique-search-cli: Create, search, and list unique items.

usage: ek_unique_cli.py command [options]

commands:
  create      Create a new unique item.
  search      Search items by name substring and/or tag.
  list        List all stored items.
```

### Environment variable: `DATA_FILE`

Set this to control storage location (default: `./data/items.json`). Example:

```bash
export DATA_FILE=$HOME/my_unique_items.json
```

## Commands & Examples

All commands below assume `python ek_unique_cli.py ...` (or `./ek_unique_cli.py ...` if executable).

### Create an item with autogenerated name

```bash
python ek_unique_cli.py create
```

Output Example:
```
Item created:
ID: 91284a7e-a3b7-4929-bfcd-2c42c7b81089
  Name: item-91284a7e
  Tags: -
  Created: 2024-03-14T10:05:07.928272
```

### Create an item with manual name and tags

```bash
python ek_unique_cli.py create --name "myalpha" --tags "foo,urgent,project1"
```

Output Example:
```
Item created:
ID: 2bae4fdc-963d-468d-9746-b9b20970d4bb
  Name: myalpha
  Tags: foo, project1, urgent
  Created: 2024-03-14T10:07:44.540078
```

**Note:** If the name is a duplicate (case-insensitive), you will see:
```
Error: Item with name 'myalpha' already exists.
```

### Search for items by name substring

```bash
python ek_unique_cli.py search --name "alp"
```

Output (if matches exist):
```
ID: 2bae4fdc-963d-468d-9746-b9b20970d4bb
  Name: myalpha
  Tags: foo, project1, urgent
  Created: 2024-03-14T10:07:44.540078
```

If no matches:
```
No items found matching criteria.
```

### Search for items by tag

```bash
python ek_unique_cli.py search --tag "foo"
```

Output:
```
ID: 2bae4fdc-963d-468d-9746-b9b20970d4bb
  Name: myalpha
  Tags: foo, project1, urgent
  Created: 2024-03-14T10:07:44.540078
```

You can also combine `--name` and `--tag` to filter on both.

### List all items

```bash
python ek_unique_cli.py list
```

Example output:
```
ID: 91284a7e-a3b7-4929-bfcd-2c42c7b81089
  Name: item-91284a7e
  Tags: -
  Created: 2024-03-14T10:05:07.928272
--
ID: 2bae4fdc-963d-468d-9746-b9b20970d4bb
  Name: myalpha
  Tags: foo, project1, urgent
  Created: 2024-03-14T10:07:44.540078
```

If no items are present:
```
No items found.
```

### Error handling
- Attempts to create an item with a duplicate name (case-insensitive) will fail with an error.
- Attempts to write when disk is full or file permissions are bad will show a clear error.
- If data file is corrupted, a backup is created and the tool starts fresh.
- Invalid tags (tags with commas or control characters) will be rejected politely.

## Deployment

Local usage only. To migrate data, simply copy your data file.

## Development
- Each code module is isolated and testable.
- Tests are in `tests/` and run with pytest.
- Data is always stored atomically with locking for safety.
- You can set/unset `DATA_FILE` for different runs or test setups.
